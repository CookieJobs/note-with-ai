# noteWithAI 项目说明

本项目包含前端（Next.js）与后端（Node.js/Express）两部分，以及若干维护脚本。

## 贡献约定（必须遵守）
- 任何功能、架构、写法的更新，工作结束后必须同步更新：
  - 受影响目录的 `README.md`（目录架构 3 行以内 + 文件清单：名字/地位/功能）
  - 相关代码文件的开头注释（3 行：`Input` / `Output` / `Pos`，并附更新声明）
- 提交前将运行文档校验，若存在缺失或占位未完善，将拒绝提交。

## 仓库结构（概览）
- `frontend/`：Next.js 前端应用
- `backend/`：Node.js/Express 后端服务
- `scripts/`：项目维护与运维脚本

## 文档与注释规范
- 目录级 README：
  - 用 1–3 行描述目录职责与架构定位
  - 列出“文件名字 / 地位 / 功能”清单
  - 末尾声明：一旦我所属的文件夹有所变化，请更新我
- 文件级头注释（代码文件）：
  - `Input`：依赖外部的什么
  - `Output`：对外提供什么
  - `Pos`：在系统局部的地位是什么
  - `Note`：一旦我被更新，务必更新我的开头注释，以及所属的文件夹的 README

## 脚本与校验
- 提供 `doc:scan`、`doc:inject`、`doc:check` 三个脚本，用于扫描、注入模板与校验规范。
- 可选集成 `husky` 的 `pre-commit` 钩子，在提交前自动执行 `doc:check`。
