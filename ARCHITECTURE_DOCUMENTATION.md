## 1. 项目概述

### 1.1 产品定位

**NoteWithAI** 是一款**AI 驱动的智能笔记应用**，核心特点是：

- ✍️ **富文本编辑**: 支持 TipTap 编辑器，提供专业的写作体验
- 🤖 **AI 自动摘要**: 自动生成笔记标题、关键词、语义摘要
- 🔗 **语义联想**: 基于向量搜索，发现笔记之间的关联关系
- 💬 **AI 对话**: 基于 DeepSeek 的智能聊天助手
- 📊 **个性化推荐**: 根据用户笔记内容推荐相关文章

### 1.2 核心价值

```
传统笔记应用          vs          NoteWithAI
─────────────────────────────────────────────────
手动整理笔记          →          AI 自动摘要
孤立的笔记存储        →          语义关联发现
简单的搜索功能        →          向量语义搜索
单一的文字记录        →          富文本 + AI 对话
```

## 4. 数据库设计

### 4.1 MongoDB 数据模型

#### 4.1.1 用户表 (Users)

```typescript
// 文件位置: backend/models/User.ts

{
  _id: ObjectId,              // 主键
  username: String,           // 用户名 (唯一, 2-20字符)
  email: String,              // 邮箱 (唯一)
  password: String,           // 密码 (bcrypt加密, 最小8字符)
  avatar: String,             // 头像 URL
  isActive: Boolean,          // 账户状态
  createdAt: Date,            // 创建时间
  updatedAt: Date             // 更新时间
}

// 索引设计
// - username: 唯一索引
// - email: 唯一索引
```

**数据关系**: 一个用户 → 多条笔记、多条聊天会话

#### 4.1.2 笔记表 (Notes)

```typescript
// 文件位置: backend/models/Note.ts

{
  _id: ObjectId,              // 主键
  userId: ObjectId,           // 外键 → Users._id
  content: String,            // 笔记纯文本内容 (必填)
  contentJson: Mixed,         // TipTap 富文本 JSON 结构 (可选)
  contentText: String,        // 派生纯文本，用于搜索/embedding
  title: String,              // 笔记标题 (AI 生成或首行提取)
  summary: String,            // 语义摘要 (AI 生成，≤120字，用于联想)
  concepts: [String],         // 概念扩展词 (AI 生成，8-12个)
  recommendCache: Mixed,      // 推荐结果缓存 (节省 LLM token)
  keywords: [String],         // 关键词 (AI 生成，1-5个)
  embedding: [Number],        // 向量 (Qwen 生成，1024维)
  createdAt: Date,            // 创建时间
  updatedAt: Date             // 更新时间
}

// 索引设计
// - userId: 普通索引 (按用户查询)
// - userId + embedding: 复合索引 (向量查询优化)
// - createdAt: 普通索引 (时间排序)
// - title, summary, content, contentText: 全文索引
// - keywords: 普通索引 (关键词搜索)
```

**数据关系**: 一个用户 → 多条笔记

#### 4.1.3 聊天会话表 (Chats)

```typescript
// 文件位置: backend/models/Chat.ts

{
  _id: ObjectId,              // 主键
  userId: ObjectId,           // 外键 → Users._id
  title: String,              // 对话标题 (AI 生成或默认)
  messages: [{
    role: String,             // 角色: 'user' | 'assistant'
    content: String           // 消息内容
  }],
  createdAt: Date,            // 创建时间
  updatedAt: Date             // 更新时间 (用于排序)
}

// 索引设计
// - userId: 普通索引 (按用户查询)
// - userId + updatedAt: 复合索引 (用户+时间排序)
// - createdAt: 普通索引 (时间排序)
// - updatedAt: 普通索引 (最近更新)
// - title: 全文索引 (标题搜索)
```

**数据关系**: 一个用户 → 多个聊天会话 → 多个消息

### 8.2 相似度计算

```typescript
// 文件位置: backend/utils/embedding.ts

// 余弦相似度计算
export function cosineSimilarity(a: number[], b: number[]): number {
  if (a.length !== b.length || a.length === 0) return 0;
  
  const dotProduct = a.reduce((sum, val, i) => sum + val * b[i], 0);
  const magnitudeA = Math.sqrt(a.reduce((sum, val) => sum + val * val, 0));
  const magnitudeB = Math.sqrt(b.reduce((sum, val) => sum + val * val, 0));
  
  return magnitudeA && magnitudeB ? dotProduct / (magnitudeA * magnitudeB) : 0;
}

// 使用示例
const similarity = cosineSimilarity([0.1, 0.2, 0.3], [0.1, 0.2, 0.3]);
// similarity = 1.0（完全相同）
```

### 8.3 多路召回算法

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         多路召回算法流程                                  │
└─────────────────────────────────────────────────────────────────────────┘

1. 查询构造
   q0: "标题 + 正文" → 获取/生成向量
   q1: "AI 摘要"     → 获取/生成向量
   q2: "概念词"      → 获取/生成向量

2. 并行向量查询
   ┌─────────────────────────────────────────────────────────────┐
   │                      并行执行                               │
   │                                                             │
   │   q0 向量 ──→ 召回 TopK1 ──┐                               │
   │                          ├──→ 合并、去重、分数取max        │
   │   q1 向量 ──→ 召回 TopK2 ──┤                               │
   │                          │                                 │
   │   q2 向量 ──→ 召回 TopK3 ──┘                               │
   │                                                             │
   └─────────────────────────────────────────────────────────────┘
            │
            ▼
   ┌─────────────────────────────────────────┐
   │   候选池（去重后）                       │
   │   - 计算每条笔记的 s1max（多路最高分）   │
   │   - 记录各路得分 s1q[0], s1q[1], s1q[2]  │
   └─────────────────────────────────────────┘
            │
            ▼
   ┌─────────────────────────────────────────┐
   │   TopN 筛选                             │
   │   - 按 s1max 排序                       │
   │   - 取 TopK（finalK）                   │
   └─────────────────────────────────────────┘

3. LLM 重排
   ┌─────────────────────────────────────────┐
   │   TopK 候选笔记详情获取                  │
   │   - 获取标题、摘要、内容                 │
   │   - 检查推荐缓存                         │
   └─────────────────────────────────────────┘
            │
            ├──────────────────────────────────┐
            ▼                                  ▼
   ┌─────────────────┐              ┌─────────────────────────┐
   │  缓存命中       │              │  缓存未命中             │
   │  直接使用缓存   │              │  调用 DeepSeek LLM      │
   └─────────────────┘              │  - 语义相关性评分（s2） │
                                    │  - 关联类型分类         │
                                    │  - 推荐理由生成         │
                                    └─────────────────────────┘
                                                  │
                                                  ▼
                                    ┌─────────────────────────┐
                                    │   分数融合              │
                                    │   score = 0.3*s1 + 0.7*s2│
                                    └─────────────────────────┘
                                                  │
                                                  ▼
                                    ┌─────────────────────────┐
                                    │   阈值筛选 & 排序       │
                                    │   hardThreshold = 0.62  │
                                    └─────────────────────────┘
                                                  │
                                                  ▼
                                    ┌─────────────────────────┐
                                    │   最终推荐结果          │
                                    └─────────────────────────┘
```

## 11. 数据流与业务逻辑

### 11.1 笔记创建完整流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         笔记创建数据流详解                                │
└─────────────────────────────────────────────────────────────────────────┘

1. 用户操作

2. 前端处理
   ┌─────────────────────────────────────────────────────────────┐
   │  输入验证                                                    │
   │  - contentText 不能为空                                      │
   │  - contentJson 必须是对象类型                                │
   │                                                             │
   │  数据格式化                                                  │
   │  - 提取首行作为 fallback title                               │
   │  - 纯文本 contentText                                        │
   │  - 富文本 contentJson (如果有)                               │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   POST /api/notes

3. 后端处理
   ┌─────────────────────────────────────────────────────────────┐
   │  认证检查                                                    │
   │  - 验证 JWT Token                                            │
   │  - 提取用户信息                                              │
   │                                                             │
   │  数据验证                                                    │
   │  - 内容非空检查                                              │
   │  - contentJson 类型检查                                      │
   │                                                             │
   │  数据保存                                                    │
   │  - 创建 Note 文档                                            │
   │  - title: 首行文本（或"未命名笔记"）                         │
   │  - content: 纯文本                                          │
   │  - contentText: 纯文本                                      │
   │  - contentJson: 富文本 JSON (如果有)                         │
   │  - userId: 用户 ObjectId                                     │
   │  - summary/concepts/keywords/embedding: 空（后续异步生成）  │
   │  - recommendCache: null                                      │
   └─────────────────────────────────────────────────────────────┘
          │
          ├──────────────────────────────────────────┐
          ▼                                          ▼
   ┌─────────────────┐                  ┌─────────────────────────┐
   │  返回响应       │                  │  后台异步任务           │
   │  201 Created    │                  │  不阻塞主流程           │
   └─────────────────┘                  └─────────────────────────┘
                                                │
                                                ├──→ 任务1: 生成向量
                                                │   ├── 读取笔记
                                                │   ├── 调用 Qwen Embedding
                                                │   └── 更新 embedding 字段
                                                │
                                                ├──→ 任务2: AI 元数据
                                                │   ├── 调用 DeepSeek API
                                                │   ├── 生成 title/keywords
                                                │   ├── 生成 summary
                                                │   └── 生成 concepts
                                                │
                                                └──→ 任务3: 更新缓存
                                                    └── recommendCache
```

### 11.2 语义联想完整流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         语义联想业务逻辑详解                              │
└─────────────────────────────────────────────────────────────────────────┘

1. 触发条件
   - 用户在笔记详情页点击"联想"按钮
   - 或在聊天中查看相关笔记推荐

2. 请求发起
   POST /api/recommend/semantic-notes
   {
     "noteId": "507f1f77bcf86cd799439011",
     "recallK": 30,      // 召回数量
     "finalK": 10,       // 最终数量
     "s1Threshold": 0.35, // 向量召回阈值
     "hardThreshold": 0.62 // 最终阈值
   }

3. 后端处理 - 第一阶段：数据准备
   ┌─────────────────────────────────────────────────────────────┐
   │  获取当前笔记                                                │
   │  - 通过 noteId + userId 验证所有权                            │
   │  - 提取 title, content, summary, concepts, embedding         │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  构造查询向量                                                │
   │                                                             │
   │  q0 = "标题 正文"                                            │
   │  → 有 embedding? 复用 : 调用 Qwen 生成                       │
   │                                                             │
   │  q1 = "摘要" (如果存在)                                      │
   │  → 有 embedding? 复用 : 调用 Qwen 生成                       │
   │                                                             │
   │  q2 = "概念词拼接" (如果存在)                                 │
   │  → 有 embedding? 复用 : 调用 Qwen 生成                       │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   并行查询各路向量相似度
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  多路召回合并                                                │
   │                                                             │
   │  对每路查询结果：                                            │
   │  - 计算余弦相似度                                            │
   │  - 过滤 threshold 以下的记录                                 │
   │  - 按 score 排序，取 TopK                                     │
   │                                                             │
   │  合并策略：                                                  │
   │  - 使用 Map 去重（按 _id）                                    │
   │  - s1max = 多路最高分                                        │
   │  - s1q = 各路得分数组                                        │
   │                                                             │
   │  结果：候选池（pool）                                        │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  TopK 筛选                                                  │
   │  - 按 s1max 排序                                            │
   │  - 取 TopK（finalK）                                        │
   │                                                             │
   │  查询详情                                                    │
   │  - 批量获取 TopK 笔记的详细信息                              │
   │  - title, summary, contentText, updatedAt                   │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  缓存命中检查                                                │
   │                                                             │
   │  检查条件：                                                  │
   │  - recommendCache 存在                                      │
   │  - algoVersion === 'semantic-notes-v3'                      │
   │  - sourceUpdatedAt === 当前笔记 updatedAt                   │
   │  - 候选笔记 updatedAt 未变化                                 │
   │                                                             │
   │  命中处理：                                                  │
   │  - 直接使用缓存的 s2, type, reason                           │
   │  - 记录 cacheHits++                                         │
   │                                                             │
   │  未命中：                                                    │
   │  - 加入 missing 数组                                         │
   │  - 记录 cacheMisses++                                       │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  LLM 重排（仅 missing 候选）                                 │
   │                                                             │
   │  输入：                                                      │
   │  - current: 当前笔记信息                                     │
   │  - candidates: missing 数组                                  │
   │                                                             │
   │  DeepSeek 调用：                                            │
   │  - system prompt: 联想任务说明                               │
   │  - user prompt: JSON 格式的 current + candidates             │
   │                                                             │
   │  输出：                                                      │
   │  - s2: 0-1 的相似度评分                                      │
   │  - type: 强相关/补充背景/延伸阅读/类比/因果/反例/弱关联       │
   │  - reason: 中文短句（15-25字）                               │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  分数融合                                                    │
   │                                                             │
   │  计算公式：                                                  │
   │  score = 0.3 * s1max + 0.7 * s2                              │
   │                                                             │
   │  说明：                                                      │
   │  - s1: 向量召回阶段分数（快速、粗糙）                         │
   │  - s2: LLM 重排阶段分数（精确、深度语义）                     │
   │  - LLM 权重更高（0.7）因为语义理解更准确                     │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  阈值筛选                                                    │
   │                                                             │
   │  过滤条件：                                                  │
   │  - score >= hardThreshold (0.62)                             │
   │                                                             │
   │  排序：                                                      │
   │  - 按 score 降序                                            │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  缓存写入（异步，不阻塞返回）                                 │
   │                                                             │
   │  更新 recommendCache 字段：                                  │
   │  - algoVersion                                              │
   │  - sourceUpdatedAt                                          │
   │  - generatedAt                                              │
   │  - params                                                   │
   │  - byCandidateId                                            │
   │    - s2                                                     │
   │    - type                                                   │
   │    - reason                                                 │
   │    - candidateUpdatedAt                                     │
   │    - cachedAt                                               │
   └─────────────────────────────────────────────────────────────┘
          │
          ▼
   返回结果
   {
     "recommendations": [...],
     "meta": {...}
   }

4. 前端展示
   - 渲染推荐笔记卡片
   - 显示关联类型图标
   - 显示推荐理由
   - 显示相似度分数（可选）
```

## 12. 扩展性考虑

### 12.1 水平扩展

```
当前架构支持水平扩展：

┌─────────────────────────────────────────────────────────────────────────┐
│                         水平扩展方案                                     │
└─────────────────────────────────────────────────────────────────────────┘

1. 前端扩展
   - Vercel 自动扩展 Serverless 实例
   - CDN 静态资源分发
   - 无状态设计，易于负载均衡

2. 后端扩展
   - 无状态 Express 应用
   - 可以部署多个实例
   - 使用 Nginx / K8s 进行负载均衡

3. 数据库扩展
   - MongoDB Atlas 支持自动分片
   - 读写分离（Replica Set）
   - 考虑引入 Redis 缓存层

4. AI 服务扩展
   - DeepSeek API 本身已具备扩展能力
   - Qwen Embedding API 也具备扩展能力
   - 本地可以增加请求队列和限流
```

### 12.2 功能扩展点

```
易扩展的功能模块：

1. 新的 AI 服务商
   - 文件: backend/services/deepseek.ts
   - 可添加 OpenAI / Anthropic / 文心一言等
   - 只需实现相同的接口方法

2. 新的向量数据库
   - 文件: backend/utils/embedding.ts
   - 可替换为 Pinecone / Milvus / Weaviate
   - 只需修改向量生成和相似度计算逻辑

3. 新的编辑器
   - 文件: frontend/src/app/notes/components/
   - 可替换为 ProseMirror / Slate / Draft.js
   - 保持 JSON 格式兼容即可

4. 新的搜索服务
   - 文件: backend/services/search.ts
   - 可添加 Elasticsearch / Algolia
   - 保持接口一致

5. 缓存层扩展
   - 可引入 Redis 替代内存缓存
   - 支持分布式部署
   - 提高缓存持久性
```

### 12.3 性能优化建议

```
当前性能瓶颈及优化建议：

1. 向量生成
   - 当前：异步生成，单条调用
   - 优化：批量生成 + 队列处理

2. LLM 调用
   - 当前：实时调用
   - 优化：增加请求缓存 + 限流
```
