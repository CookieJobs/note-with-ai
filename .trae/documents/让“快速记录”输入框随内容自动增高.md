## 问题概述
- 当前位置：输入框位于 Notes 页面 [page.tsx:L939-L949](file:///Users/liujin/Documents/noteWithAI/frontend/src/app/notes/page.tsx#L939-L949)，样式在 [notes.module.scss](file:///Users/liujin/Documents/noteWithAI/frontend/src/app/notes/notes.module.scss)。
- 现状：固定 `rows` 与 CSS `min-height`/`max-height`，未根据内容动态调整高度，导致多行输入时需要滚动查看之前内容。

## 方案与原理
- 在 React 中通过 `textareaRef` 读取 DOM 的 `scrollHeight`，动态设置 `style.height` 达到“随内容增高”的效果。
- 展开态：将高度先置为 `auto` 再置为 `scrollHeight`，保证计算准确。
- 折叠态：清空内联高度，继续由 `rows={1}` 与折叠样式控制。
- 保留 `max-height: 300px`，并为输入框启用 `overflow-y: auto`，当内容过多时仅在输入框内部滚动，避免页面抖动。

## 具体改动
1. 前端逻辑
- 在 `NotesContent` 中新增 `useEffect`，监听 `newContent` 与 `composeCollapsed`，动态调整高度：

```ts
useEffect(() => {
  const el = textareaRef.current;
  if (!el) return;
  if (!composeCollapsed) {
    el.style.height = 'auto';
    el.style.height = `${el.scrollHeight}px`;
  } else {
    el.style.height = '';
  }
}, [newContent, composeCollapsed]);
```

- 在 `onChange` 保持现有 `setNewContent`；无需引入第三方库。

2. 样式优化
- 在 `.composeInput` 中补充：`overflow-y: auto;`，保留 `max-height: 300px;`。
- 不更改现有字体与行高设定，移动端行为保持一致。

## 验证与回归点
- 在 Notes 页面一次性粘贴多行内容：输入框高度随内容增长，超过 300px 后仅在输入框内滚动。
- 切换折叠/展开：展开时自适应高度，折叠时恢复单行。
- 快捷保存（Cmd/Ctrl + Enter）无影响；提交后输入框重置高度与内容。

## 影响范围与风险
- 仅影响 Notes 页面“快速记录”输入体验，无跨页面影响。
- 使用内联样式高度，不破坏模块化样式；回退简单（移除 useEffect 与 `overflow-y` 即可）。
