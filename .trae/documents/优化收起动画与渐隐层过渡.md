## 目标
修复笔记收起时“没有动画”的视觉问题，并优化过渡体验。

## 问题定位
经过分析，收起时“无动画”的根本原因很可能是 **渐隐层（Fade Overlay）的瞬间出现**。
1. 当点击“收起”时，`isExpanded` 变为 `false`。
2. React 立即渲染 `fadeOverlay` 组件（之前是条件渲染）。
3. 由于 `fadeOverlay` 现在的样式是 **底部不透明的白色渐变**，它会瞬间遮挡住正文底部的内容。
4. 即使正文容器的高度在 200ms 内平滑收缩，但用户肉眼首先看到的是底部文字“瞬间消失/被遮挡”，从而产生了“没有动画”或“生硬截断”的错觉。

## 解决方案
为了解决这个问题，我们需要让渐隐层也参与动画过渡：
1. **持久化渲染**：只要内容可截断（`canExpand`），就始终渲染 `fadeOverlay`，而不是根据 `isExpanded` 进行挂载/卸载。
2. **Opacity 过渡**：通过 CSS `opacity` 属性控制渐隐层的显示与隐藏。
   - 展开时：`opacity: 0`（渐隐层慢慢消失，露出完整文字）。
   - 收起时：`opacity: 1`（渐隐层慢慢浮现，配合高度收缩）。
3. **动画同步**：调整 `fadeOverlay` 的 `transition` 时长，使其与高度变化的动画节奏一致。

## 代码改动
### 1. `frontend/src/app/notes/notes.module.scss`
- 修改 `.fadeOverlay`：
  - 增加 `opacity: 0`（默认隐藏）。
  - 增加 `transition: opacity 0.3s ease`。
  - 保持 `pointer-events: none`，防止阻挡点击。
- 新增 `.fadeOverlayVisible`：
  - 设置 `opacity: 1`。
- 优化 `.noteTextWrapper`：
  - 增加 `will-change: max-height` 以提升动画性能。
  - 将 `transition` 时长调整为 `300ms`，让动画更清晰。

### 2. `frontend/src/app/notes/page.tsx`
- 修改 `fadeOverlay` 的渲染逻辑：
  - 只要 `canExpand && !isEditingContent` 就渲染。
  - 通过 `className` 动态切换可见性：`${styles.fadeOverlay} ${!isExpanded ? styles.fadeOverlayVisible : ''}`。

## 预期效果
- **展开时**：正文高度平滑变大，同时底部的白色遮罩慢慢变透明，文字清晰浮现。
- **收起时**：正文高度平滑变小，同时底部的白色遮罩慢慢变不透明，优雅地“盖住”底部文字，消除瞬间截断感。
