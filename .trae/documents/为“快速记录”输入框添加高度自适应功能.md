我将修改 `frontend/src/app/notes/page.tsx` 文件，为“快速记录”输入框添加高度自适应功能。

### 具体实施步骤

1.  **添加自动调整高度的逻辑**：
    在 `NotesContent` 组件中引入 `useEffect` 钩子，监听输入内容 `newContent` 和折叠状态 `composeCollapsed` 的变化。

2.  **实现高度计算**：
    - 当输入框处于**展开状态**（`!composeCollapsed`）时：
        - 先将输入框高度重置为 `'auto'`，以便正确计算内容高度。
        - 然后将高度设置为 `scrollHeight`（内容实际高度），实现随内容自动拉长。
    - 当输入框处于**折叠状态**时：
        - 清除内联高度样式（设为 `''`），让组件恢复默认的 CSS 样式和 `rows={1}` 属性控制，保持单行显示。

3.  **代码变更预览**：
    ```typescript
    // 在 NotesContent 组件内添加
    useEffect(() => {
      const textarea = textareaRef.current;
      if (textarea) {
        if (!composeCollapsed) {
          // 展开时：先重置为 auto 以获取正确 scrollHeight，再设置为实际高度
          textarea.style.height = 'auto';
          textarea.style.height = `${textarea.scrollHeight}px`;
        } else {
          // 折叠时：清除内联样式，使用默认 CSS/rows 属性
          textarea.style.height = '';
        }
      }
    }, [newContent, composeCollapsed]);
    ```

这个方案不依赖额外的第三方库，轻量且高效，能够完美解决您在输入多行内容时无法完整查看上下文的问题，同时保留了输入框的最大高度限制（由 CSS `max-height: 300px` 控制），避免内容过多时占用整个屏幕。
