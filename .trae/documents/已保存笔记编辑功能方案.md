## 产品目标
- 为已保存笔记提供可编辑能力（标题、正文、关键词），在不打扰现有写作与推荐功能的前提下，提升整理与复盘效率。
- 保障数据安全（鉴权、冲突处理、版本留痕）、良好体验（自动保存、撤销、提示）、与智能联动（摘要/关键词/向量更新）。

## 现状与差距
- 已有：创建笔记与自动摘要、关键词生成（POST `/api/notes`）、列表展示与删除、标题内联编辑（POST `/api/notes/:id`）。
  - 后端：`backend/routes/notes.ts:16-22` 获取列表；`backend/routes/notes.ts:24-46` 创建；`backend/routes/notes.ts:48-62` 删除；`backend/routes/notes.ts:101-122` 仅支持“更新标题”。
  - 前端：`frontend/src/app/notes/page.tsx:401-435` 拉取列表；`frontend/src/app/notes/page.tsx:477-514` 创建；`frontend/src/app/notes/page.tsx:449-461` 删除；`frontend/src/app/notes/page.tsx:55-87` 标题保存。
- 缺少：正文与关键词的编辑；自动保存与冲突解决；版本历史；编辑后向量与推荐联动策略。

## MVP 范围（P0）
- 正文可编辑（支持基础 Markdown/纯文本）与标题内联编辑并存。
- 关键词可编辑（手动为主），可选“自动重生成”。
- 自动保存（防抖）与明确状态提示；失败可重试；基本冲突处理（基于 `updatedAt`）。
- 保存后触发向量重算任务（异步），确保聊天页相关笔记检索的准确性。

## 交互设计
- 列表页内联编辑：
  - 标题：继续沿用当前交互；按 Enter 保存、Esc 取消；展示“保存中/已保存/失败”状态。
  - 正文：卡片展开后切换为编辑模式（`textarea`/轻量 Markdown 编辑器）；顶部显示保存状态与“撤销”按钮。
  - 关键词：chip + 输入框；允许增删；提供“重新生成”按钮（调用后端 AI 摘要）。
- 详情编辑页（建议新增 `/notes/[id]`）：
  - 适合长文档：左右布局（正文编辑区 + 信息侧栏）；侧栏显示关键词、创建/更新时间、版本历史。
  - 支持快捷键（Cmd/Ctrl+S 手动保存；Cmd/Ctrl+Enter 完成并返回）。
- 状态与提示：统一 toast/inline 状态条；失败时保留本地草稿恢复。

## 后端改造
- 新增部分更新接口（更推荐 Patch 语义）：
  - `PATCH /api/notes/:id` 支持任意字段的局部更新：`{ title?, content?, keywords? }`。
  - 参数校验：长度与类型；`content` 合理大小上限；`keywords` 为字符串数组。
  - 鉴权与归属：沿用 `authenticateToken` 与 `ResourceValidator.validateOwnership`。
  - 乐观并发控制：请求携带 `updatedAt`；后端在更新前比对，冲突返回 `409 Conflict` 与最新 `note`。
  - 返回结构：`{ note }`，保持与现有风格一致（`ResponseHandler.success`）。
- Embedding 与摘要联动：
  - 当 `content` 变化：保存后排队异步触发 `POST /api/notes/:id/embed`（或后端内部队列），避免写入路径阻塞。
  - 自动摘要与关键词：提供开关参数 `autoSummarize: boolean`，为 true 时调用 `summarizeNote(content)` 覆盖 `title/keywords`。
  - 统计与维护：沿用 `backend/services/noteEmbedding.ts` 的维护与统计能力。

## 前端改造
- API 封装：新增 `patchNote(id, payload)` 封装，统一错误处理与类型。
- 列表页：
  - 正文编辑态：卡片正文切换为编辑器；本地 state 跟踪 `dirty`、`saving`、`error`；失焦或停顿 800ms 防抖保存。
  - 关键词编辑：chip 组件支持增删与回车确认；点击“重新生成”触发 AI。
  - 冲突处理：若 409，弹出“检测到他处更新”，展示差异并提供“覆盖/合并/取消”。
- 详情页：
  - 长文编辑体验（滚动性能、固定工具栏、字数统计、预览）。
  - 版本历史查看与回滚（MVP 可仅保留最近 N 次草稿快照）。

## 自动保存与离线
- 防抖保存：输入停止 800ms 发起；并发中断与队列控制，保证最终一致。
- 断网/失败：本地缓存草稿（`localStorage`/IndexedDB），恢复后提示合并。
- 明确状态：保存中（旋转图标）、已保存（打勾）、失败（错误文案+重试）。

## 权限与安全
- 严格鉴权与归属校验；限制正文与关键词的长度。
- 速率限制：编辑接口对单用户限流，防止恶意刷写。
- 审计可扩展：为版本历史预留审计记录（后续可合规化）。

## 性能与体验
- 大笔记性能：编辑器虚拟化/分段渲染（大段文本滚动顺滑）。
- 向量更新异步：请求成功后提示“正在更新智能推荐”，不阻塞前端。
- 关键词可视化：chip 排序与权重提示（增强可选）。

## 指标与观测
- 保存成功率、平均保存时延、409 冲突率、自动保存触发频次。
- 向量更新耗时与失败率；推荐页点击率变化（与编辑前后对比）。
- 错误日志聚合（前后端统一上报）。

## 验收标准
- 能在列表页与详情页编辑并保存标题/正文/关键词，自动保存稳定且有状态反馈。
- 并发编辑返回 409 并提供可用的用户决策路径；无数据丢失。
- 编辑后相关笔记推荐与聊天检索生效（在合理延迟内完成向量更新）。

## 里程碑
- P0（2-3 天）：`PATCH /api/notes/:id`、列表页正文/关键词编辑、自动保存与基本并发控制、Embedding 异步触发。
- P1（3-5 天）：详情页编辑器、版本历史快照与回滚、自动摘要开关与手动重生成。
- P2（后续）：更细致的冲突合并、富文本/Markdown 全功能、审计与可观测完善。
