## 目标
实现笔记收起时的平滑过渡动画，与展开时的动画效果一致。

## 问题分析
当前展开/收起通过切换 CSS 类名 (`.noteText` vs `.noteTextExpanded`) 实现。`.noteText` 使用了 `display: -webkit-box` 和 `-webkit-line-clamp` 来截断文本，而 `.noteTextExpanded` 使用 `display: block`。由于 `display` 属性的切换是突变的，导致 `max-height` 的 CSS transition 动画在收起时失效或表现异常。

鉴于我们已经引入了**底部渐隐层**来处理视觉截断，不再强依赖 `-webkit-line-clamp` 产生的省略号。

## 解决方案
1. **统一正文样式**：移除 `-webkit-line-clamp` 相关属性，统一使用 `display: block`。这意味着正文在 DOM 中总是完整渲染的，仅通过外层容器的 `max-height` 来限制可见区域。
2. **纯 CSS 过渡**：完全依赖外层容器 `.noteTextWrapper` 的 `max-height` 变化来实现展开/收起动画，避免内部元素布局属性突变。

## 代码改动
1. **`frontend/src/app/notes/notes.module.scss`**：
   - 修改 `.noteText`，移除 `-webkit-line-clamp`、`-webkit-box-orient`、`display: -webkit-box` 等属性。
   - 删除或合并 `.noteTextExpanded`，使其样式与 `.noteText` 一致（或者直接复用）。
   - 确保 `.noteTextWrapper` 保留 `transition: max-height ...`。

2. **`frontend/src/app/notes/page.tsx`**：
   - 在渲染正文 `<p>` 标签时，不再根据 `isExpanded` 切换类名，始终使用 `styles.noteText`。
   - 保持现有的 `useEffect` 逻辑，该逻辑负责根据展开状态计算并设置精确的 `maxHeight` 像素值。

## 预期效果
- 点击“展开”：高度从约 132px 平滑增长到全文高度。
- 点击“收起”：高度从全文高度平滑收缩回约 132px。
- 无论展开还是收起，渐隐层和按钮的显隐逻辑保持不变。
