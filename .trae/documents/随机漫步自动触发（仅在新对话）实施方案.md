## 方案要点
- 保持“随机漫步”面板自身按挂载自动拉取（含防重/中止与错误重试）
- 在聊天页中仅在“新建且空会话”时挂载面板，其他场景不挂载，从而实现自动触发且不干扰交互

## 改动范围
- 前端仅修改 `frontend/src/app/chat/page.tsx`
  - 新增本地状态 `showCare` 和引用 `lastActivatedSessionIdRef`
  - 监听 `currentSession.id` 与 `currentSession.messages.length`
  - 触发逻辑：
    - 当存在会话且消息数为 0，且该会话未激活过“随机漫步”，设置 `showCare = true`，记录已激活会话 ID
    - 当消息数 > 0 时，设置 `showCare = false`
  - 渲染：仅在 `showCare === true` 时渲染 `CareAssistantPanel`
- 保持 `CareAssistantPanel` 当前实现（挂载时自动拉取 + AbortController + 防重 + 错误提示与重试）；无需再通过 ref 主动触发

## 交互保障
- 不干扰输入：面板只在消息为空的新会话显示；一旦用户发送消息即隐藏
- 不重复触发：每个会话仅记录一次激活；会话 ID 切换（`local_` → 服务端 ID）后也不会重复显示（使用已激活记录）
- UI 兼容：不修改现有组件结构与样式，仅控制是否渲染

## 错误与健壮性
- 面板内部请求失败时不影响输入与发送；提供“重试”按钮
- 组件卸载时中止网络请求，避免状态闪烁

## 验证场景
- 首次进入应用：若创建新会话且为空，自动显示面板并拉取
- 创建新对话：立即显示并拉取；用户发送首条消息后隐藏
- 从历史会话切换：历史会话已有消息，不显示面板；切回空会话时显示一次
- 网络不稳定：面板提示失败并支持重试；不影响输入框

## 交付
- 只改 `chat/page.tsx` 的挂载控制逻辑，代码简单、风险低；现有面板逻辑无需改动，符合原设计的流畅体验
