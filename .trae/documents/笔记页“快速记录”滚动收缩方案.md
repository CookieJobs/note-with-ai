## 背景与目标
- 背景：在笔记页面，顶部的“快速记录”区域（textarea + 操作区）始终占据固定高度；当用户浏览历史笔记时希望该区域在用户滚动时适当收缩，提升列表可视空间。
- 目标：当用户在列表区域向下滚动达到阈值时，“快速记录”视觉缩小；当用户点击/聚焦继续记录或返回顶部时恢复正常高度。

## 现状与定位
- 入口文件：`frontend/src/app/notes/page.tsx`，快速记录容器见 `composeArea`（734-794），列表滚动容器见 `notesScrollContainer`（796-813）。
- 样式文件：`frontend/src/app/notes/notes.module.scss`，包含 `.composeArea`、`.composeActions`、`.submitButton` 等相关样式。

## 技术实现
- 新增状态
  - `composeCollapsed: boolean` 用于记录是否进入收缩态；仅在 `!isComposing` 时允许收缩。
- 监听滚动
  - 为 `notesScrollContainer` 添加 `ref`（如 `notesScrollRef`），在 `useEffect` 中绑定 `scroll` 事件，基于 `scrollTop > 阈值(如 80px)` 设置 `composeCollapsed`。
  - 使用 `requestAnimationFrame`/轻量节流，避免滚动频繁触发导致性能抖动。
- 交互规则
  - 聚焦快速记录 `textarea` 时：强制取消收缩（`composeCollapsed=false`，`isComposing=true`）。
  - 取消编辑或失焦后：若滚动位置仍超过阈值，重新进入收缩态。
  - 滚动回到顶部（`scrollTop <= 阈值`）：恢复正常态。
- 结构与样式调整
  - `composeArea` 增加条件类：`composeCollapsed ? styles.composeAreaCollapsed : ''`。
  - `textarea` 行数动态：`rows={composeCollapsed ? 1 : 3}`；在样式中为 `.composeAreaCollapsed .composeInput` 设置更小的 `min-height/padding/font-size`。
  - 操作提示与按钮：在收缩态弱化或隐藏 `.composeActions`（如 `opacity: 0; pointer-events: none; height: 0;` 或折叠为一行提示），保留主要提交按钮在正常态；聚焦时恢复显示。
- 动画与过渡
  - 为 `composeArea`/`composeInput` 的 `padding`、`min-height`、`font-size`、`line-height` 添加 `transition: 200ms ease`，实现平滑收缩/展开。

## 兼容性与性能
- 兼容：移动端断点下（768px 以下）确保收缩态不影响可点击区域；必要时在小屏降低阈值（如 40px）。
- 性能：滚动监听基于单一容器 + 轻节流，避免多次布局抖动；在 React 18 StrictMode 下确保 `useEffect` 绑定/解绑正确。

## 验证用例
- 桌面端：滚动超过阈值 → 收缩生效；滚动回顶部 → 恢复正常；点击 `textarea` → 展开；取消编辑 → 若仍超过阈值继续收缩。
- 移动端：小屏在滚动与输入法弹出情况下，收缩/展开行为不影响可用性。
- 无笔记与大量笔记两种数据量下，视觉与交互一致。
