# 笔记嵌入维护工具指南

本文档介绍了笔记嵌入系统的维护工具。该工具用于扫描数据库中缺失向量的笔记，并自动修复它们（生成向量）。

**注意**: 目前系统已实现"保存笔记时立即生成向量"的功能，因此本工具主要作为**维护和修复工具**使用，用于处理网络异常等情况导致的向量生成失败。

## 🎯 系统概述

笔记嵌入维护系统包含以下核心组件：

- **repair_embeddings.ts**: 维护脚本，负责扫描并修复缺失的笔记嵌入
- **noteEmbedding.ts**: 嵌入生成服务，提供批量处理和统计功能
- **embedding.ts**: 配置文件，包含所有相关参数设置

## ✨ 功能特性

- 🛠️ **手动修复**: 随时运行，修复缺失的向量数据
- 📊 **批量处理**: 支持大量笔记的批量嵌入生成
- 📈 **统计监控**: 提供嵌入生成统计和进度跟踪
- 🛡️ **错误处理**: 完善的重试机制和错误恢复
- ⚡ **性能优化**: 缓存机制和批处理优化

## 🔧 环境要求

### 系统要求
- Node.js >= 16.0.0
- MongoDB >= 4.4
- TypeScript >= 4.5

### 必需的API密钥
- **DASHSCOPE_API_KEY**: 阿里云DashScope API密钥（用于生成嵌入向量）

## 🚀 使用方法

### 1. 安装依赖

```bash
# 进入后端目录
cd backend

# 安装依赖
npm install
```

### 2. 配置环境变量

确保 `.env` 文件中配置了数据库连接和API密钥：

```bash
# 数据库配置
MONGODB_URI='mongodb://localhost:27017/noteWithAI'

# API密钥配置
DASHSCOPE_API_KEY='your_dashscope_api_key_here'
```

### 3. 运行修复工具

当发现有笔记无法被搜索到，或者需要批量修复历史数据时，运行以下命令：

```bash
# 运行修复工具
npm run repair:embeddings
```

该命令会：
1. 连接数据库
2. 扫描所有没有 embedding 字段的笔记
3. 批量调用 AI 接口生成向量
4. 保存回数据库
5. 输出处理报告

## 🔍 故障排除

### 常见问题

#### 1. 环境变量缺失
```
错误: 缺少必要的环境变量: DASHSCOPE_API_KEY
解决: 检查.env文件中是否正确配置了所有必需的环境变量
```

#### 2. 数据库连接失败
```
错误: MongoDB连接失败
解决: 
- 确保MongoDB服务正在运行
- 检查MONGODB_URI配置是否正确
```

#### 3. API调用失败
```
错误: DashScope API调用失败
解决:
- 检查API密钥是否有效
- 确认账户余额充足
- 检查网络连接
```

## 📞 技术支持

如果在使用过程中遇到问题，请：

1. 检查日志输出获取详细错误信息
2. 确认所有环境变量和依赖配置正确
3. 验证API密钥和数据库连接
