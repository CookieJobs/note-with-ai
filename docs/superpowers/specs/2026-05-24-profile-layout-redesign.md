# 个人中心布局重设计

**日期**: 2026-05-24
**状态**: 已确认
**关联**: `/profile` 页面

## 问题

1. **布局不平衡** — 两栏布局中左侧堆叠 6 张卡片，右侧只有「每日推荐」
2. **更新画像含义不明** — 按钮放在用户信息卡片底部，用户不清楚点击后会影响哪些卡片

## 设计方案

### 整体结构

保持两栏布局，内容区顶部新增横向统计条。左栏按功能拆分为两个明确分组。

```
┌─ TopNavigation ──────────────────────────────┐
├─ 横向统计条（笔记数·本月·连续天·字数·标签·分析时间）─┤
├──────────────────┬───────────────────────────┤
│  账户设置         │                           │
│  ├ 用户信息       │  每日推荐 (For You)         │
│  └ 编辑/密码      │  ├ 最新动态                │
│                  │  ├ 温故知新                │
│  AI 画像  🔄     │  └ ...                    │
│  ├ 兴趣图谱       │                           │
│  ├ 技能领域       │                           │
│  ├ 个人传记       │                           │
│  └ 氛围色         │                           │
└──────────────────┴───────────────────────────┘
```

### 关键设计决策

| 决策 | 方案 |
|------|------|
| 统计面板位置 | 移至内容区顶部，横向一行展示，释放左栏空间 |
| 左栏分组 | 「账户设置」和「AI 画像」两个独立区块 |
| 更新画像位置 | AI 画像组标题右侧 + 副标题「基于你的笔记内容自动生成」 |
| AI 卡片视觉 | 浅紫色背景 + 紫色边框，与账户白底区分 |
| AI 画像折叠 | 支持折叠/展开，默认展开；analyzing 时折叠并显示动画 |
| 横向统计条 | 8 项横向排列；移动端水平滚动或精简为 4 项 |
| 最后分析时间 | 统计条最右侧显示 |
| 移动端 | 统计条横向滚动，单栏：账户→AI 画像→推荐 |

## 实现范围

### Frontend

1. **`profile/page.tsx`** — 重构页面结构
   - `StatsBar` 横向统计条（替代纵向 `StatsPanel`）
   - 左栏拆为 `AccountSection` 和 `AIProfileSection`
   - `AIProfileSection` 支持折叠/展开，标题右侧放「更新画像」+ 副标题

2. **`profile/profile.module.scss`** — 样式调整
   - `.statsBar` 横向统计条
   - `.sectionGroup` 分组卡片（`.accountGroup` / `.aiGroup`）
   - AI 子项浅紫背景
   - 折叠动画（max-height transition）
   - 移动端横向滚动

### Backend — 无改动

## 不变更

- 每日推荐、氛围色主题、编辑资料/修改密码 Modal、后端接口
